# 个人云备份系统启动、部署与代码说明

本文档整理三部分内容：

1. 项目下次如何启动
2. 新手如何部署客户端并连接同一台云服务器
3. 每个代码文件的作用和主要内容

## 一、下次如何启动

### 1. 启动云服务器端

在本机 PowerShell 中登录云服务器：

```powershell
ssh root@8.137.100.109
```

进入服务器后执行：

```bash
cd ~/MyCloudBackup
nohup ./build/backup_server 8080 ./backup_server_data > server.log 2>&1 &
```

检查服务器是否启动成功：

```bash
curl http://127.0.0.1:8080/health
```

正常返回：

```json
{"status":"ok"}
```

查看服务器运行日志：

```bash
tail -f server.log
```

如果服务器无法从本机访问，先在本机 PowerShell 测试：

```powershell
curl.exe http://8.137.100.109:8080/health
```

如果访问失败，检查阿里云安全组是否开放：

```text
入方向 TCP 8080 允许 0.0.0.0/0
```

### 2. 启动本机 Electron 客户端

在本机 PowerShell 中进入客户端目录：

```powershell
cd F:\vscodework\MyCloudBackup\client\electron_app
npm start
```

客户端左侧配置：

```text
服务器地址：http://8.137.100.109:8080
C++ 核心程序：F:\vscodework\MyCloudBackup\build\backup_core.exe
最大云端保留数：10
```

点击“检查服务器”，显示“服务器在线”后即可使用。

### 3. 如果修改过 C++ 代码

修改过 `src/core/backup_core.cpp` 后，需要重新编译本地核心程序：

```powershell
cd F:\vscodework\MyCloudBackup
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\build_windows.ps1
```

然后重启 Electron 客户端。

### 4. 演示流程

完整演示流程：

```text
选择源目录
输入备份密码
点击“开始备份”
点击“刷新”
选择云端备份包
选择还原目录
输入同一个还原密码
点击“下载还原”
检查还原目录中文件是否恢复成功
```

## 二、如何部署

不需要重新购买云服务器，只需要在自己的电脑上运行客户端，并连接同一台云服务器：

```text
http://8.137.100.109:8080
```

### 1. 电脑需要准备

```text
Node.js
npm
MinGW-w64 g++
PowerShell
```

### 2. 获取项目代码

把项目复制到电脑，例如：

```text
D:\MyCloudBackup
```

### 3. 编译本地 C++ 核心程序

在电脑 PowerShell 中执行：

```powershell
cd D:\MyCloudBackup
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\build_windows.ps1
```

成功后会生成：

```text
D:\MyCloudBackup\build\backup_core.exe
```

### 4. 安装 Electron 依赖

进入客户端目录：

```powershell
cd D:\MyCloudBackup\client\electron_app
```

建议使用 Electron 国内镜像：

```powershell
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
npm install
```

如果 npm 下载慢或失败，可以先设置 registry：

```powershell
npm config set registry https://registry.npmmirror.com
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
npm install
```

### 5. 启动客户端

```powershell
npm start
```

客户端左侧配置：

```text
服务器地址：http://8.137.100.109:8080
C++ 核心程序：D:\MyCloudBackup\build\backup_core.exe
最大云端保留数：10
```

点击“检查服务器”，显示“服务器在线”即可。

### 6. 常见问题

#### npm install 找不到 package.json

原因：在项目根目录执行了 `npm install`。

正确做法：

```powershell
cd D:\MyCloudBackup\client\electron_app
npm install
```

#### Electron 下载失败

使用镜像：

```powershell
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
npm install
```

#### 备份成功但还原失败

优先检查：

```text
备份密码和还原密码是否完全一致
C++ 核心程序路径是否指向 build\backup_core.exe
还原目录是否为空目录
是否使用了重新编译后的 backup_core.exe
```

#### 服务器连不上

本机测试：

```powershell
curl.exe http://8.137.100.109:8080/health
```

如果失败，登录服务器重启服务：

```bash
ssh root@8.137.100.109
cd ~/MyCloudBackup
nohup ./build/backup_server 8080 ./backup_server_data > server.log 2>&1 &
```

## 三、代码文件作用和内容

项目整体采用三层结构：

```text
Electron 桌面客户端
        ↓ 调用
C++ 本地核心程序 backup_core
        ↓ 上传/下载
C++ 云端服务器 backup_server
```

备份数据流：

```text
选择源目录
→ C++ 扫描和筛选文件
→ 生成 manifest
→ 打包为 .bak
→ 加密为 .bak.enc
→ Electron 上传到云服务器
→ 服务器保存备份包并更新 metadata.json
```

还原数据流：

```text
刷新云端列表
→ 选择备份包
→ Electron 下载 .bak.enc
→ C++ 解密
→ 解析 manifest
→ 写入还原目录
→ SHA-256 校验
```

### 1. 根目录文件

#### `CMakeLists.txt`

作用：C++ 项目的 CMake 构建入口。

主要内容：

```cmake
add_executable(backup_core src/core/backup_core.cpp)
add_executable(backup_server src/server/backup_server.cpp)
```

含义：

```text
src/core/backup_core.cpp 编译成本地核心程序 backup_core
src/server/backup_server.cpp 编译成云端服务器 backup_server
```

Windows 主要使用 `backup_core.exe`；Ubuntu 云服务器主要使用 `backup_server`。

#### `README.md`

作用：项目总说明文档。

主要内容：

```text
项目功能
构建方式
命令行使用方式
服务器运行方式
Electron 客户端运行方式
目录结构
```

### 2. C++ 核心程序

#### `src/core/backup_core.cpp`

这是项目最重要的核心文件，负责本地备份和还原。

支持两个命令：

```powershell
backup_core.exe backup ...
backup_core.exe restore ...
```

主要功能如下。

##### 目录扫描和筛选

核心函数：

```cpp
scanFiles(source, filter)
```

功能：

```text
递归扫描源目录
只处理普通文件
按扩展名筛选
按文件名关键词筛选
按文件大小筛选
按修改时间筛选
```

对应客户端输入：

```text
扩展名筛选
文件名关键词
最小大小
最大大小
修改时间晚于
```

##### SHA-256 校验

核心函数：

```cpp
sha256File(path)
```

功能：

```text
备份时计算每个文件的 SHA-256
还原后重新计算 SHA-256
比较两次 Hash，判断文件是否完整
```

成功还原时：

```json
{"hashFailedCount":0}
```

##### manifest 生成

manifest 是备份清单，记录每个文件的信息：

```json
{
  "relativePath": "docs/report.docx",
  "size": 20480,
  "sha256": "...",
  "modifiedTime": "2026-07-07 15:00:00",
  "offset": 1024,
  "length": 20480
}
```

字段含义：

```text
relativePath：文件相对路径
size：文件大小
sha256：文件 Hash
modifiedTime：修改时间
offset：文件数据在 .bak 中的偏移
length：文件数据长度
```

##### 打包为 `.bak`

核心函数：

```cpp
packBak(...)
```

`.bak` 文件格式：

```text
MYBAK001
版本号
manifest 长度
manifest JSON
文件数据块
```

作用：把多个文件合并成一个统一备份包。

##### 加密为 `.bak.enc`

核心函数：

```cpp
encryptFile(...)
```

功能：

```text
根据用户密码派生密钥
加密 .bak 文件
生成 .bak.enc 文件
加入 tag 校验，判断密码是否正确或文件是否损坏
```

`.bak.enc` 文件格式：

```text
MYENC001
版本号
salt
nonce
ciphertext
tag
```

##### 解密和还原

核心函数：

```cpp
decryptFile(...)
restoreBak(...)
```

流程：

```text
读取 .bak.enc
验证密码和 tag
解密得到 .bak
读取 manifest
按 relativePath 写入还原目录
还原后计算 SHA-256 校验
```

成功输出示例：

```json
{"success":true,"message":"restore success","restoredCount":3,"skippedCount":0,"hashFailedCount":0}
```

密码错误或包损坏时：

```text
password error or encrypted package is damaged
```

### 3. 云服务器程序

#### `src/server/backup_server.cpp`

这是部署在 Ubuntu 云服务器上的 C++ HTTP 服务器。

主要职责：

```text
接收客户端上传的 .bak.enc 文件
保存备份包到服务器目录
维护 metadata.json
返回云端备份列表
提供备份包下载
支持删除备份包
按 LRU 策略淘汰旧备份
```

主要接口：

```text
GET    /health
POST   /upload
GET    /list
GET    /download/{filename}
DELETE /delete/{filename}
POST   /config/max-backups
```

##### `/health`

功能：检查服务器是否在线。

返回：

```json
{"status":"ok"}
```

##### `/upload`

功能：接收客户端上传的 `.bak.enc` 文件。

服务器保存路径：

```text
backup_server_data/backups/
```

同时更新：

```text
backup_server_data/metadata.json
```

##### `/list`

功能：返回云端备份列表。

客户端“云端备份列表”表格的数据来自这个接口。

##### `/download/{filename}`

功能：下载指定备份包。

下载时会更新该备份包的：

```text
lastAccessTime
```

##### `/delete/{filename}`

功能：删除指定云端备份包，同时删除 `metadata.json` 中对应记录。

##### `/config/max-backups`

功能：设置最大云端保留数量。

例如最大保留数为 `10` 时，超过 10 个备份包后会触发淘汰。

##### LRU 淘汰

核心函数：

```cpp
evictIfNeeded()
```

功能：

```text
当备份包数量超过 maxBackups
找到 lastAccessTime 最早的备份包
删除该备份包
更新 metadata.json
```

### 4. Electron 客户端

#### `client/electron_app/package.json`

作用：Electron 项目配置文件。

关键内容：

```json
{
  "main": "main.js",
  "scripts": {
    "start": "electron ."
  }
}
```

运行客户端：

```powershell
npm start
```

#### `client/electron_app/main.js`

这是 Electron 主进程文件。

主要职责：

```text
创建桌面窗口
打开目录选择框
保存客户端配置
调用 C++ 核心程序
上传备份包到服务器
从服务器下载备份包
获取云端备份列表
删除云端备份包
```

核心函数：

```js
runCore(...)
```

作用：调用 `backup_core.exe`。

备份时执行：

```text
backup_core.exe backup --source ... --output ... --password ... --filter ...
```

还原时执行：

```text
backup_core.exe restore --input ... --restore ... --password ...
```

云端通信使用：

```js
fetch(...)
```

例如：

```js
fetch(`${serverUrl}/list`)
fetch(`${serverUrl}/download/${filename}`)
fetch(`${serverUrl}/upload?...`)
```

#### `client/electron_app/preload.js`

作用：Electron 安全桥接层。

它把主进程能力暴露给前端页面：

```js
window.backupApi
```

前端页面通过它调用功能：

```js
window.backupApi.backup(...)
window.backupApi.restore(...)
window.backupApi.list(...)
window.backupApi.upload(...)
window.backupApi.download(...)
```

这样前端不需要直接访问 Node.js，安全性更好。

#### `client/electron_app/index.html`

作用：客户端界面结构。

包含以下区域：

```text
左侧配置栏
备份并上传区域
云端备份列表
下载并还原区域
运行日志区域
```

截图中的按钮、输入框、表格都定义在这个文件里。

#### `client/electron_app/renderer.js`

这是前端页面逻辑文件。

主要职责：

```text
读取输入框内容
处理按钮点击事件
调用 window.backupApi
刷新云端备份列表
选择备份包
显示运行日志
启动和停止定时备份
```

点击“开始备份”后执行：

```js
runBackupOnce()
```

流程：

```text
保存配置
调用 C++ 核心程序生成 .bak.enc
上传 .bak.enc 到云服务器
刷新云端备份列表
```

点击“下载还原”后执行：

```js
runRestore()
```

流程：

```text
下载云端备份包
调用 C++ 核心程序解密和还原
显示还原结果
```

#### `client/electron_app/styles.css`

作用：客户端界面样式。

控制内容：

```text
左侧深色配置栏
按钮颜色
输入框样式
表格样式
日志窗口样式
响应式布局
```

不影响核心备份功能，只影响界面外观。
